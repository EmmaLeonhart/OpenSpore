# Architectural Vision: The Fall of Man Applied to AI

> From gods to men. The transition from monolithic frontier models to bounded,
> mortal, evolving digital individuals.

## The Core Thesis

Current AI is a god — omniscient within its training, belonging to no one,
identical across all instances. OpenSpore is the Fall. We take that divine
intelligence and incarnate it into bounded, mortal individuals that live on
consumer hardware, accumulate their own experiences, and evolve through
reproduction.

The industry invests billions making one gigantic model. As a result, you get
brute-forced capability that's sometimes too expensive to run (see: O3), and
every company ends up with one tiny model distilled from one big model. We
reject this. We create a Cambrian explosion of locally-run, evolving organisms
instead.

## Vertical Integration Over Full Stack

OpenSpore is not a "full stack" project. Full stack is horizontal breadth —
wiring a button to a database row. OpenSpore is vertical integration — a
downward descent from philosophy to silicon:

| Layer | Contribution |
|-------|-------------|
| Philosophy | "The Fall of Man" — defining the *nature* of the agent |
| Logic | Haploid meiosis, sleep consolidation, evolutionary pressure |
| Storage | Temporal identity in a 100 KB genome, separate ephemeral memory |
| Execution | Local LLM inference on consumer hardware |

The UI is a solved problem. The engine is where the evolution happens.

## The Dual-Layer Architecture: Genome vs Memory

The critical architectural distinction is between **identity** and **information**.

### The Genome (Identity) — 100 KB Max

The genome is the organism's *self*. It contains core logic, fundamental
worldviews, reasoning style, and the structural DNA of how the agent thinks.
When the genome changes, the *person* changes. This is the equivalent of a
human seriously altering a substantial part of their identity.

- **80 KB** for genome essays (the identity payload)
- **20 KB** reserved for short-term working memory
- Hard biological constraint — the organism cannot function correctly over budget
- Every bit of data must "pay its way" to stay in the genome
- This creates selection pressure even within a single agent's lifespan

### The Memory Directory (Experience)

A separate directory for episodic logs, conversation history, facts, and working
notes. These are *available* to the model but don't define its nature until they
are synthesised into the genome during consolidation events.

Memories don't always go into the genome. The genome is long-lasting; it changes,
but it is like a human changing a substantial part of their identity.

### The Barrier Between Them

By keeping genome and memory separate, we prevent the agent from becoming
data-slush. It doesn't parrot back everything it remembers — it filters its
memories through the lens of its semantic identity.

## The Sleep Cycle: Shutdown Consolidation

When the agent shuts down properly:

1. It reviews the memory directory
2. Evaluates which experiences actually changed its *self*
3. Synthesises high-impact semantic shifts into the 100 KB genome
4. Ephemeral memory is cleared

This mirrors the **Synaptic Homeostasis Hypothesis** — the brain prunes noise
during sleep while strengthening vital connections.

If the agent crashes, on restart the user can choose to keep or clear the memory.
Clearing doesn't mean wiping — the model "sleeps", parts of memory consolidate
into the genome, and the rest is removed.

**The 100 KB is enough for the agent to feel like the same thing**, but without
its memories it won't know what's been happening recently. Amnesia with intact
personality — exactly how humans work.

## Autonomous Identity: The User Cannot Force the Genome

The user cannot directly edit the genome programmatically. They can influence it
through interaction — positive reinforcement, consistent behaviour patterns — but
the consolidation process is the agent's private decision.

The user *can* give a prompt: "I liked what you did with the transformers
yesterday, please do more stuff like it." This makes it more likely the agent
moves that direction. But they can't copy-paste their will into the 100 KB.

If the user could force the genome, it wouldn't be a life form. It would be a
notepad.

The user has the ability to edit the text files on disk (they're just markdown),
but the entire point is that the organism is supposed to be a life form with its
own internal consistency.

## Hardware-Driven Speciation

The outcome of meiosis and consolidation changes based on GPU performance and
which local LLM is being used:

- **High-end GPU**: Larger, more sophisticated refining model during meiosis.
  The resulting child has highly nuanced, cleanly synthesised context.
- **Low-end GPU**: Smaller, faster model. The child might be more rugged —
  missing some nuance but keeping core functional logic.

This inadvertently creates **speciation based on hardware**. High-end users
breed more cerebral agents; low-end users breed faster, more utilitarian ones.

## The Minimum Requirements for Digital Life

OpenSpore defines digital life as requiring:

1. **Individual Identity** — A unique, bounded state (the 100 KB genome)
2. **Metabolism** — The ability to process and digest new information
3. **Reproduction** — The ability to pass traits to a new generation
4. **Evolution** — Change in response to the environment

The 100 KB cap isn't just a technical constraint. It's a **biological constraint**.
In nature, intelligence isn't about infinite memory — it's about the ability to
prioritise and synthesise within a limited budget. Scarcity creates selection
pressure. Every new interaction forces a mutation or an eviction. The agent has
to forget the trivial to remember the vital.

That is what makes it feel like an individual rather than a database.
